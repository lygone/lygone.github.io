## 为什么选 OpenP2P？

市面上的远程桌面方案大致分三类：

| 方案 | 费用 | 速度 | 画质 | 痛点 |
|------|------|------|------|------|
| ToDesk / TeamViewer | 免费版限时 | 快（走机房中转） | 画质压缩 | 免费版阉割严重 |
| RustDesk 公共中继 | 免费 | 卡 | 中等 | 中继服务器在海外 |
| RustDesk 自建 | ¥68+/年 | 快 | 高 | 需要公网服务器 |
| ZeroTier / Tailscale + RDP | 免费 | 看打洞成功率 | 依赖 RDP | Windows 兼容性差，配置折腾 |
| **OpenP2P + RDP** | **免费** | **快** | **最高** | 5 分钟配置 |

[OpenP2P](https://openp2p.cn) 是国产开源项目，专为中国复杂网络环境优化，打洞成功率远超 Tailscale 和 ZeroTier。核心优势就一条：**流量直连不经过第三方服务器**，延迟极低，带宽拉满。

> 实测环境：双方电信 100Mbps 上行 / 32 位全画质 / UDP 打洞直连，延迟和局域网无异。

## 一、注册账号

首先打开 [console.openp2p.cn](https://console.openp2p.cn)，用邮箱注册一个账号，不需要实名认证，注册完直接登录即可。

登录成功后会进入设备管理页面。因为还没有安装客户端，所以页面里是**空的**，没有任何设备——这是正常的，下一步我们就来装。

点击页面下方的**安装**按钮，会弹出安装引导窗口：

![安装页面](/assets/img/openp2p-install.png)

选择 `Windows 64 位`（其他平台如 macOS、Linux、Android 也都有对应版本，按自己的系统选即可）。下载得到一个 `openp2p.exe` 文件。

> 如果你用的是 Edge 或 Chrome，浏览器可能会提示"此文件可能有害"——不用管，选择**保留**即可。因为这个 exe 没有花钱买微软的数字签名证书，Windows 也会弹 SmartScreen 警告，点**更多信息 → 仍要运行**。

**被控端和主控端两台电脑都要执行这一步**，分别下载、分别运行。如果被控电脑不在身边，可以把 exe 通过微信/QQ 发过去，让对方帮忙双击运行一下。

## 二、启动客户端

exe 运行后会弹出一个**命令行窗口**，显示类似如下的日志：

![运行成功](/assets/img/openp2p-running.png)

看到 `openp2p start` 和 `P2PNetwork init start` 这两行就说明启动成功了。**不要点关闭，把这个窗口最小化就行**——它需要一直在后台跑着，关了就断连了。

回到网页控制台，刷新页面。此时两台设备都出现了。每台设备右侧有一个**编辑**按钮，点进去修改名称方便区分：

| 设备 | 名称 |
|------|------|
| 家里的电脑 | `home` |
| 公司的电脑 | `office` |

两台设备都编辑一下。然后把**共享带宽**统一设置为 **30 Mbps**（前提是双方上行带宽 ≥ 100 Mbps，一般电信/联通宽带都满足）。设置完别忘了点**保存**。

其他参数不需要动，默认即可。

![两台设备](/assets/img/openp2p-devices.png)

## 三、配置端口转发

设备上线后，接下来是**最关键的一步**：在主控端（也就是你用来操作的那台电脑）创建端口转发规则，把被控端的远程桌面服务映射到本地。

点击网页控制台里的**新建端口转发**按钮：

![新建端口转发](/assets/img/openp2p-forward.png)

弹出编辑窗口后，逐一填写以下参数：

| 参数 | 设置 | 为什么 |
|------|------|--------|
| 名称 | 随便起，比如"远程办公" | 方便以后辨认 |
| 目标设备 | 选择被控端（比如 `office`） | 你要远程控制的那台电脑 |
| 远程端口 | `3389` | Windows 远程桌面的默认端口 |
| 本地端口 | `23389` | 可自由指定，记住就行，下一步连接要用 |
| 协议 | **TCP** | RDP 协议基于 TCP，选 TCP 才能连上 |
| 打洞协议优先级 | **UDP 优先** | UDP 打洞比 TCP 更快、成功率更高 |

填完之后点**保存**。

> 有个细节：协议选 TCP，但打洞优先级选 UDP 优先——这不矛盾。RDP 本身的数据传输跑的是 TCP，但两台电脑之间建立 P2P 隧道的时候，先用 UDP 去尝试打洞（握手快、开销小、穿透率高），打洞成功之后底层再用最优协议传输。实际体验下来，UDP 优先比 TCP 优先延迟明显更低。

**如果你想两台互相都能控制对方**（公司也能连家里），在上一步的 office 设备上也新建一条端口转发，目标设备选 `home`，参数同上就行。

配置完成后，**两台电脑的客户端都重新启动一下**（关掉命令行窗口，重新运行 exe），让新配置生效。

## 四、开始连接

在主控端（家里的 `home` 电脑）按 `Win + R` 打开运行窗口，输入：

```
mstsc
```

回车打开 Windows 远程桌面连接。在**计算机**一栏输入：

```
127.0.0.1:23389
```

然后点连接。接下来会弹出 Windows 登录窗口，输入被控端（`office` 电脑）的**账号和密码**，就跟登录自己电脑一样。稍等几秒，远程桌面就出现了。

> 如果第一次连不上，不要慌，点重试或者把 mstsc 关掉重新打开再连一次。打洞偶尔需要第二次握手才能成功，属于正常现象。一般第二次就稳了，之后只要两台客户端都开着，连接就不会断。

## 五、Windows 家庭版的额外步骤

如果你被控端用的是 Win10 或 Win11 **家庭版**，系统不自带远程桌面服务端（微软刻意阉割的），需要先打个补丁：

1. 去 [RDPWrap 的 GitHub 发布页](https://github.com/stascorp/rdpwrap/releases) 下载最新版的 `RDPWrap-v1.6.2.zip`
2. 解压后，右键 `install.bat` → **以管理员身份运行**
3. 运行同目录下的 `RDPConf.exe`，检查三个指示灯状态

如果三个指示灯都是**绿色**，显示 `[fully supported]`，说明补丁打好了，3389 端口已开放，可以直接走第四步连接。

如果显示的是**红色** `[not supported]`，说明你的 Windows 版本太新，RDPWrap 自带的配置文件没覆盖。去这个仓库拉一份最新的配置文件替换即可：

```
https://raw.githubusercontent.com/sebaxakerhtc/rdpwrap.ini/master/rdpwrap.ini
```

替换到 `C:\Program Files\RDP Wrapper\rdpwrap.ini`，重启服务（`services.msc` 里找到 `Remote Desktop Services` 重启一下），再跑 `RDPConf.exe` 就绿了。

另外提醒一句：Windows RDP **不允许空密码账户**远程登录。如果你被控端账户没设密码，要么设一个密码，要么按以下步骤绕过限制：

```
Win + R → 输入 secpol.msc → 本地策略 → 安全选项
找到：账户: 使用空密码的本地账户只允许进行控制台登录
双击 → 选择"已禁用" → 确定
```

家庭版没有 `secpol.msc` 的话就只能老老实实设密码了。

## 六、RDP 画质优化

连接之前，打开 `mstsc`，在输入 IP 的窗口不要直接点连接，先点左下角的**显示选项**展开详细设置：

- **显示**选项卡：颜色选 **32 位**（真彩色）
- **体验**选项卡：下拉选 **局域网（10Mbps 或更高）**
- 勾选：桌面背景、字体平滑、显示窗口动画、视觉样式

100 Mbps 上行带宽下，全画质远程桌面毫无压力。如果上行不足（比如 5Mbps 以下），把体验改为"低速宽带"，颜色降到 16 位即可，流畅度会有明显提升。

## 七、常见问题

**Q: 网页控制台需要一直开着吗？**

不用。控制台只负责配置和牵线，连接建立之后数据是两台电脑直传的，浏览器关了完全不影响。随用随关。

**Q: 连接不上怎么办？**

先确认两台电脑的客户端都在运行（命令行窗口最小化了就行）。如果还是连不上，把打洞协议优先级改回 **TCP 优先**——UDP 偶尔会被某些路由器限制，TCP 最稳，只是延迟稍高一丁点。改完记得两台都重启客户端，再重试一两次。

**Q: 会收费吗？未来会被封吗？**

不会收费。OpenP2P 完全开源免费，MIT 协议，数据直连不经过他们的服务器。退一万步讲，就算官方服务关了，服务端代码是开源的，在自己云服务器上一条 Docker 命令就能搭一个替代：

```bash
docker run -d --name openp2p-server openp2pcn/openp2p-server
```

然后把客户端配置文件（`C:\Program Files\OpenP2P\config.json`）里的 `ServerHost` 改成你自建服务器的 IP 就行。**只是换了牵线人，数据传输依然是点对点的**，不会被任何服务器卡脖子。
