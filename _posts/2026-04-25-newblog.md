博客基于 [Chirpy](https://github.com/cotes2020/jekyll-theme-chirpy/) 主题搭建，托管在 GitHub Pages。从零到上线大约花了三个小时，以下是完整流程。

## 第一步：创建仓库

访问 Chirpy 项目主页，使用页面右上角的 **Use this template** → **Create a new repository** 创建你自己的仓库。仓库名必须是：

	你的用户名.github.io

例如我的就是 `lygone.github.io`。创建后 GitHub Pages 会自动生效，此时访问 `https://你的用户名.github.io` 应该能看到默认首页了。

## 第二步：本地克隆并修改配置

用 GitHub Desktop 或 `git clone` 把仓库拉到本地。核心配置都在 `_config.yml` 这一个文件里，逐项对照填写：

	title: 你的博客名                  # 网页标题
	tagline: 你的副标题                 # 侧边栏小字
	description: 博客描述               # SEO 用
	url: "https://你的用户名.github.io"   # 站点地址
	avatar: /assets/img/avatar.jpg     # 头像路径
	timezone: Asia/Shanghai            # 时区
	lang: zh-CN                        # 语言

	github:
	  username: 你的GitHub用户名

	social:
	  name: 你的名字
	  email: 你的邮箱
	  links:
	    - https://github.com/你的用户名

## 第三步：写文章

文章放在 `_posts` 文件夹里，文件名格式必须严格遵守：

	YYYY-MM-DD-文章标题.md

每篇文章开头要有 YAML 头信息（front matter）：

	---
	layout:     post
	title:      文章标题
	tags:       标签1 标签2
	category:   分类名
	description: 文章摘要
	---

配图放在 `assets/img/` 目录下，文章中这样引用：

	![图片描述](/assets/img/你的图片.jpg)

## 第四步：提交并等待部署

所有改动提交到 GitHub 后，GitHub Actions 会自动构建部署，一般 1-2 分钟完成。如果改了之后没看到新文章：

	Ctrl + Shift + R 强制刷新（清除缓存）

页面底部会弹出「发现新版本内容」提示，点击更新即可。

## 第五步：配置评论

评论系统用的是 [Giscus](https://giscus.app/)，利用 GitHub Discussions 存储评论。配置步骤：

1. 打开仓库的 Settings → General → Features，勾选 **Discussions**
2. 访问 [giscus.app](https://giscus.app/)，按指引填写仓库名
3. 拿到四个参数填入 `_config.yml` 的 `comments.giscus` 区域：

	repo: 你的用户名/你的用户名.github.io
	repo_id: "MDEwOl..."
	category: "Announcements"
	category_id: "DIC_kw..."

4. 重新提交部署，评论框就出来了。

从零开始到评论功能全部跑通大约三小时。后续坚持更新，主要写游戏相关内容和一些技术笔记。
